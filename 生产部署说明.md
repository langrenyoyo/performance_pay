# 绩效工资系统生产部署说明

## 系统形态

当前系统已改为数据库模式：

1. 前端页面：`index.html`、`app.js`、`styles.css`
2. 后端服务：`server.py`
3. 数据库：SQLite，默认文件为 `data/performance.db`
4. 启动脚本：`start-db-server.ps1`

数据不再依赖浏览器本地存储，而是通过后端 API 保存到 SQLite 数据库。

## 启动方式

在项目目录执行：

```powershell
.\start-db-server.ps1
```

默认监听：

```text
0.0.0.0:5175
```

内网访问示例：

```text
http://192.168.110.13:5175/index.html
```

## 生产配置

可通过环境变量调整：

```powershell
$env:PERF_HOST = "0.0.0.0"
$env:PERF_PORT = "5175"
$env:PERF_DB_PATH = "D:\member\data\performance.db"
python server.py
```

## 数据库备份

定期备份以下文件：

```text
data/performance.db
data/performance.db-wal
data/performance.db-shm
```

建议在无人使用时备份，或先停止服务再复制数据库文件。

## API

健康检查：

```text
GET /api/health
```

读取某月数据：

```text
GET /api/state?period=2026-05
```

写入某月数据：

```text
PUT /api/state
Content-Type: application/json
```

请求体：

```json
{
  "period": "2026-05",
  "employees": []
}
```

加载测试数据：

```text
GET /api/state?period=2026-05&seed=1
```
