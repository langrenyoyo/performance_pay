# 绩效工资系统生产部署说明

## 系统形态

当前系统已支持生产数据库模式：

1. 前端页面：`index.html`、`app.js`、`styles.css`
2. 后端服务：`server.py`
3. 推荐生产数据库：PostgreSQL
4. 本地备用数据库：SQLite，仅用于开发或临时演示
5. 容器部署：`Dockerfile`、`docker-compose.yml`

## 推荐启动方式：Docker Compose

在项目目录执行：

```powershell
docker compose up -d --build
```

默认访问：

```text
http://127.0.0.1:5175/index.html
```

内网访问示例：

```text
http://192.168.110.13:5175/index.html
```

首次加载测试数据：

```text
http://192.168.110.13:5175/index.html?seed=1
```

## 数据库配置

生产环境通过 `DATABASE_URL` 连接 PostgreSQL：

```text
postgresql://performance:change_me@postgres:5432/performance_pay
```

本地直接运行时可设置：

```powershell
$env:DATABASE_URL = "postgresql://performance:change_me@localhost:5432/performance_pay"
python server.py
```

如果未设置 `DATABASE_URL`，系统会自动退回 SQLite 开发模式，数据库文件为：

```text
data/performance.db
```

## 依赖安装

非容器运行时安装依赖：

```powershell
pip install -r requirements.txt
```

## PostgreSQL 备份

Docker Compose 模式下备份：

```powershell
docker compose exec postgres pg_dump -U performance performance_pay > performance_pay_backup.sql
```

恢复：

```powershell
Get-Content .\performance_pay_backup.sql | docker compose exec -T postgres psql -U performance performance_pay
```

## API

健康检查：

```text
GET /api/health
```

读取某月数据：

```text
GET /api/state?period=2026-05
```

写入某月数据：

```text
PUT /api/state
Content-Type: application/json
```

请求体：

```json
{
  "period": "2026-05",
  "employees": []
}
```

加载测试数据：

```text
GET /api/state?period=2026-05&seed=1
```
